## Structure

The documentation is structured in the following way:

- Explanations: explanations of the core features of the boilerplate, including the reasoning behind the choices and how to remove a feature if not needed
- Guidelines: guides to add a feature to the boilerplate
- Tutorials: in depth guides to create a specific feature, such as a CRUD entity, a feature module, etc.

## Language

For now, the documentation is written in English and French at mininum.
