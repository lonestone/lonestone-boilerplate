## ЁЯЪА Project Structure

Inside of your Astro + Starlight project, you'll see the following folders and files:

```
.
тФЬтФАтФА public/
тФЬтФАтФА src/
тФВ   тФЬтФАтФА assets/
тФВ   тФЬтФАтФА content/
тФВ   тФВ   тФЬтФАтФА docs/
тФВ   тФФтФАтФА content.config.ts
тФЬтФАтФА astro.config.mjs
тФЬтФАтФА package.json
тФФтФАтФА tsconfig.json
```

Starlight looks for `.md` or `.mdx` files in the `src/content/docs/` directory. Each file is exposed as a route based on its file name.

Images can be added to `src/assets/` and embedded in Markdown with a relative link.

Static assets, like favicons, can be placed in the `public/` directory.

## ЁЯзЮ Commands

All commands are run from the root of the project, from a terminal:

| Command                   | Action                                           |
| :------------------------ | :----------------------------------------------- |
| `pnpm install`             | Installs dependencies                            |
| `pnpm dev`             | Starts local dev server at `localhost:4321`      |
| `pnpm build`           | Build your production site to `./dist/`          |
| `pnpm preview`         | Preview your build locally, before deploying     |
| `pnpm astro ...`       | Run CLI commands like `astro add`, `astro check` |
| `pnpm astro -- --help` | Get help using the Astro CLI                     |

## Index

The index is generated by the `generate:index` script. It is automatically updated when you add or remove a documentation file.

It contains a list of all the documentation files, and their description. Useful to quickly navigate to the documentation you need, especially for AI Agents.
